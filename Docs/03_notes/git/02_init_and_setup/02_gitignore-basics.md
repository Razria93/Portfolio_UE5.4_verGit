## Gitignore 사용법
#### 1. Gitignore 초기화

1. .gitignore 파일 생성함
2. gitignore 문법에 맞게 파일을 작성함
   
#### 2. Gitignore 문법
```Bash
# ---
# 형식 : (파일명).(확장자)
# 현재 .gitignore가 위치한 디렉토리와 그 하위 모든 경로에서
# 이름이 file.c 인 파일/디렉토리를 재귀적으로 무시
# 재귀적으로 무시한다는 것은 현재 디렉토리뿐만 아니라 
# 그 아래의 모든 하위 폴더들(하위의 하위까지 전부)을 포함해서 무시하는 것
file.c

# ---
# 형식 : /(파일명.확장자)
# 줄이 '/' 로 시작하면, 이 .gitignore가 있는 디렉토리를 기준으로 한 '절대경로'로 해석됨
# .gitignore가 저장소 루트에 있으면, 저장소 루트 기준 절대경로처럼 동작
/file.c      # (루트/.gitignore 기준) 루트의 file.c만 무시
/src/file.c  # (루트/.gitignore 기준) 루트/src/file.c만 무시

# ---
# 형식 : *.(확장자)
# 이 .gitignore가 적용되는 디렉토리 이하 모든 경로에서
# 확장자가 .c 인 파일/디렉토리를 재귀적으로 무시
# * 는 '와일드카드'로 모든 문자를 의미하는 문자임
*.c

# ---
# 형식 : !(파일명.확장자)
# 앞선 ignore 규칙에 의해 이미 무시된 파일/디렉토리 중
# not_ignore_this.c 는 예외로 다시 추적하라는 뜻
# (중요) .gitignore 규칙은 위에서 아래 '순서대로' 적용됨
!not_ignore_this.c

# ---
# 형식 : (이름)
# logs 라는 이름의 '파일 또는 디렉토리'를 무시
# 디렉토리라면 그 디렉토리와 그 내부 전체도 함께 무시됨
logs

# ---
# 형식 : (폴더명)/
# logs 라는 이름의 '디렉토리'와 그 내부 전체를 무시
# 파일 이름이 logs 인 경우는 매칭되지 않음
logs/


# ---
# 형식 : (폴더트리)/(파일명.확장자)
# .gitignore 위치 기준 상대경로, 재귀 X
# logs 폴더 바로 아래의 debug.log 와 .c 파일들을 무시
logs/debug.log   # logs/debug.log 한 개만
logs/*.c         # logs/ 바로 아래의 *.c (logs/sub/ 아래는 해당 X)

# ---
# 형식 : (폴더트리)/**/(파일명.확장자)
# logs 폴더 아래의 모든 depth에서 debug.log 무시
# '**' 는 0개 이상의 디렉토리를 재귀적으로 매칭
logs/**/debug.log
```
